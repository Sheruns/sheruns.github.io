    Hello, 这位baby



    虽然有些仓促简单，但是最后还是弄好了。

    下面是如何食用这个网站的说明：

    1. 概要

    • http://www.sheruns.me/是这个网站的地址。
    • 你的 文稿 文件夹下有一个文件名为 Sheruns 。这就是你的网站文件夹——接下来所有操作进行的地方。
    • 你所需要编辑的所有文件均为 Markdown 文件，使用标准Markdown语法编写，可以插入emoji表情，如：❤️💗💓😘

    2. 关于

    相信冰雪聪明的你已经注意到这个页面了，在页面顶部有个关于tab卡。点击它会跳转到个人介绍页面。
    打开 Sheruns/source/about 编辑其中的 index.md 即可更改个人介绍页面的内容。（记得更改后保存）

    3. 发表文章

    打开 Sheruns/source/_posts ，你会看到两个.md(Markdown文件)，这个文件夹里保存着你所有的文章(目前为2篇)，其中howtouse.md即为本食用指南。你可以随便打开一个看看里面都写了什么。

    打开后，你已经发现这和你平时做笔记所用的Markdown语法一样，但是有一部分不一样：你发现每一个文章顶部都有一段文字，像这样：

    很好，英语专业的本能使你好像明白了这段文字的意思：分别写明了这篇文章的标题、日期、和标签。

    “很简单嘛，不用你解释了”

    你这样想着，想要跃跃欲试开始写自己的第一篇文章了。

    于是，首先你在 Sheruns/source/_posts 下新建了一个.md文件，我们姑且命名为 Hello.md。

    聪明的你把之前在其他.md文件中顶部那几行字，复制到新建的 Hello.md 文件中，然后改了改：

    然后在下面洋洋洒洒地用 Markdown 语法写了几百字。我相信吃过无数次不保存的亏的你，肯定会在写完之后按下 command+s。

    “咻，终于写完了” 大功告成后的你这样想着。

    你兴奋的打开Chrome，输入www.sheruns.me，敲击回车。但是发现文章列表里并没有你刚刚写的处女作——《Sheruns来啦》。疑惑和不解在你的脸上出现了——“我辛辛苦苦敲的文章呢？”，我喜欢你困惑的样子。

    你想了想，似乎也有点明白：“我刚刚只是在我的电脑上新建编辑了文章，应该还没有同步到网站上吧，可能还需要一个同步的步骤”

    3.1 同步

    没错，接下来你需要用 终端 做一些酷酷的事情了。😯

    第一步

    你 双击command，打开了搜索框，输入terminal，打开了终端
    首先你需要输入：

    看着这行又臭又长的代码你开始丧失了耐心，其实这句话很简单。
    cd 意为“change directory”，cd XX 意为 “去 XX 文件夹”。这句话的意思是“去Sheruns文件夹”

    很显然，这段代码的打字成本并不高在cd两个字母上，而是后面那一串字符。其实有个快速的方法，就是你在打完cd两个字母和空格后，你在Finder中找到 Sheruns 文件夹，并把它拖到 终端 中。神奇的事情出现了，那串又臭又长的代码自动粘贴到了 cd 的后面。

    输入完成后按下回车。



    第二步

    输入：

    hexo 不需要知道是什么意思，g 意为“generate”，这句话的作用是把你编辑的markdown格式文章，渲染成网站上样子。

    输入完成后按下回车。

    你看到一行一行的代码不断的在终端出现，你真棒，那是你刚刚对它的命令出现的效果，它正在generate。

    “什么时候才算generate完成呢”

    你一边想一边等着，你发现过了一会终端里的代码不再动了，并在$字符后出现了灰色长方形的输入光标。这说明已经渲染完毕了。



    最后一步

    输入：

    d 意为“deploy”，英语专业的好处又一次显现出来，这句话的意思是将你在本地的页面部署到远程的网站上。做完这一步就大功告成了。

    输入完成后按下回车。

    又开始一行行代码出现了，经过第二步，相信你已经对这种情况不再感到惊讶了，最后像第二步一样，停在了$字符后出现了灰色长方形的输入光标。

    “这就是已经执行完部署的意思吧。”你这样想着。自信地像个老程序员。



    现在你打开Chrome，输入www.sheruns.me，并按command+R强制刷新了几遍。

    你的《Sheruns来啦》出现在了文章列表的第一个，现在你已经成功发表了一篇文章，让所有人都能在sheruns.me看到。

    你做到了！

    别忙着得意，虽然只有简单的三部，但是每一次 发表/编辑/删除 文章都要进行同步，我们现在回顾一下，整个同步过程分为三部：

    • cd XXXXXXX (到Sheruns文件夹中)
    • hexo g generate 渲染
    • hexo d deploy 部署

    4. 删除/修改文章

    现在已经有人开始看你的网站了。虽然你的男朋友写的很好，你很爱他，我觉得你也肯定不会把这篇文章留在你的文章列表中。

    “我已经熟练发布文章了，我想删掉这篇尴尬的教程。”

    要删掉文章很简单，进入 Sheruns/source/_posts 中，相信你已经会抢答了，那么这个文件夹不用再解释了，你写的全部文章都在这里了。

    你只需要找到你要删除的文章.md，把他扔到废纸篓中。很好，这篇文章已经不在你的电脑里了，但是它还在网上。

    相信你也知道接下来应该怎么做了——你需要和云端的文件同步一下。

    同步不用多说了吧，那三步：

    • cd
    • hexo g
    • hexo d
    • 利用部署的时间想一下这篇文章的作者💓

    修改文章也是一样，修改.md文件后，利用终端进行同步就可以了。

    5. 本地预览

    讨厌返工的你可能像我一样写md的时候想看看在网站中实时效果是什么样，一边写文章，一边看效果，这样也减少了最后发布后的错误率，在没同步到网站之前就能修改好文章。

    你当然可以这么做。但是和 同步 一样需要用到 终端。你可能现在看到终端这两个字就头疼，没事，这次只要两步。

    第一步

    打开终端并输入

    cd到Sherun文件夹，这一步和同步的第一毛一样。

    第二步

    接着输入

    server你肯定知道是服务器的意思，这句话的意思是“打开本地服务器”。

    好了，又开始一行行代码出现在终端中，不妨在这个间隙想一下这篇文章的作者。

    最后代码不增加了，停在了：

    不愧是要过专八的人，这两句话对你来说就像读母语一样简单：

    于是你打开Chrome浏览器，地址栏输入http://localhost:4000/

    如你所想的一样，你的网站出现在了http://localhost:4000/

    现在你可以一边编辑你的markdown文件(编辑后command+S保存)，一边刷新这个页面来看实时效果。

    在你编辑万全，准备正式发布这篇文章时，你自然而然地想到需要进行同步了。

    你看了一眼终端，想到刚刚它提示你——“按 Ctrl+C 来停止.”

    你按下了ctrl+c，本地预览结束了，命令行恢复如初，http://localhost:4000/ 也变成了无法访问。

    然后你像之前一样进行同步就可以正式发布到www.sheruns.me了。
